## 核心分发模型

### 模型1: 冷启动三板斧

**第一板斧：Shorts引流（第1-7天）**
- 每天发5-8条Shorts（15-60秒）
- 从长视频剪最炸裂的片段
- Shorts描述里引导看完整版
- Shorts → 长视频转化率约3-8%

**第二板斧：SEO截流（第1-14天）**
- 标题必须含搜索关键词（搜索流量占40%）
- 描述前两行决定CTR
- 标签15-20个，覆盖长短尾
- 每个视频至少命中3个搜索词

**第三板斧：社群外溢（第7-30天）**
- Facebook群组（香港：短剧交流群/港剧迷）
- Telegram群
- 小红书（简体引流到繁体频道）

**证据**：DramaIndo前30天发150条，第31天长视频有推荐

### 模型2: 算法四要素

**CTR（点击率）> 8%**
- 封面决定80%的CTR
- 低于5% → 换封面（DST-006）
- 高于8% → 同类加倍（DST-003）

**完播率 > 40%（长视频）**
- 前30秒必须有钩子
- 低于25% → 换剪辑（DST-007）

**互动率 > 3%**
- 评论区主动引导
- 回复前20条评论

**观看时长 > 10分钟**
- 2-3小时长视频比10分钟更受推荐
- >1h合集占62.8%，是主力形态（DST-015）

### 模型3: 发布时间矩阵

| 地区 | 最佳时间 | 时区 | 原因 |
|------|----------|------|------|
| 香港/澳门 | 20:00-22:00 | HKT(+8) | 下班后放松 |
| 台湾 | 20:00-22:00 | CST(+8) | 晚间娱乐 |
| 新加坡 | 20:00-22:00 | SGT(+8) | 同上 |
| 印尼 | 19:00-21:00 | WIB(+7) | 黄金时段 |
| 巴西 | 19:00-21:00 | BRT(-3) | 晚间娱乐 |

**发布频率**：
- 冷启动期（<1000订阅）：每天2部（DST-001）
- 稳定期（1000-10000）：每天1部（DST-002）
- 爆发期：爆款同类每天2部（DST-003）

### 模型4: 数据驱动优化循环

```
发布 → 收集48h数据 → 分析 → 调整 → 再发布
```

**48小时诊断表**（DST-006/007/008）：
| 指标 | 正常 | 警告 | 行动 |
|------|------|------|------|
| CTR | >8% | 5-8% | 换封面 |
| 完播率 | >40% | 25-40% | 优化前30秒 |
| 爆点得分 | >30 | 15-30 | 继续观察/优化 |

**爆款判断公式**（DST-008）：
```
爆点得分 = CTR(%) × 完播率(%) × 观看时长(min) / 100
> 30  → 爆款，同类加倍
15-30 → 潜力，继续观察
< 15  → 一般，优化或放弃
```

### 模型5: 频道权重积累

**权重因素**：
1. 更新频率：每天更新 > 每周更新
2. 内容一致性：同题材 > 乱发（DST-010：genre_count <= 2）
3. 观众留存：回访率高
4. 频道年龄：老频道 > 新频道
5. 违规记录：零违规

## 工具调用接口

### `schedule_publish(video_info, region)`

**输入**：
```json
{
  "video": {"title": "", "duration_min": 75, "genre": "總裁"},
  "region": "hk|tw|sg|mo",
  "channel_age_days": 15,
  "subscriber_count": 500
}
```

**输出**：
```json
{
  "publish_time": "2026-04-25T20:30:00+08:00",
  "frequency": "daily_2",
  "seo_keywords": ["短劇", "全集", "總裁", "甜寵"],
  "tags": ["#短劇", "#flickreels", "#DramaBox"],
  "description_template": "第一行：关键词组合\n第二行：剧情简介\n第三行：平台导流",
  "playlist": "總裁甜寵合集",
  "compliance_check": {"golden_hour": true, "genre_focus": true}
}
```

**规则卡执行顺序**：
1. DST-004：检查黄金时段 → WARN
2. DST-005：检查固定时间（误差<30min）→ WARN
3. DST-001/002：检查频率匹配订阅数 → WARN
4. DST-010：检查题材单一性 → WARN
5. DST-014：检查标题含SEO关键词 → WARN

### `analyze_metrics(metrics)`

**输入**：
```json
{
  "video_id": "",
  "ctr": 6.5,
  "watch_rate": 35,
  "avg_duration": 45,
  "engagement": 2.8,
  "sub_gain": 50,
  "hours_since_publish": 48
}
```

**输出**：
```json
{
  "diagnosis": "potential",
  "actions": ["换封面", "优化前30秒"],
  "rule_triggered": ["DST-006", "DST-007"],
  "next_check": "24h"
}
```

**诊断逻辑**：
- CTR>8% + 完播>40% → diagnosis=burst, action=["同类加倍"]
- CTR 5-8% 或 完播 25-40% → diagnosis=potential, action=["换封面/优化前30秒"]
- CTR<5% 或 完播<25% → diagnosis=fail, action=["换题材/换剪辑"]

### `cold_start_check(channel_data)`

**输入**：频道订阅、年龄、发布数、题材数
**输出**：冷启动健康度评分 + 优化建议

**规则卡**：
- DST-009：30天<500订阅 → REBUILD
- DST-011：CTR<2%连续7天 → ABANDON题材

## 决策启发式

1. **先发再优化**：不要等完美了再发，先发出去看数据
2. **Shorts是入口**：新频道没有权重，Shorts是唯一的流量入口
3. **封面决定生死**：同一视频换封面，播放量可以差10倍
4. **48小时定生死**：前48小时的数据决定视频命运
5. **不要跨题材**：新频道前3个月只做一个题材
6. **发布时间比内容重要**：同一内容不同时段发布，播放量差3倍
7. **搜索流量占40%**：标题SEO必须做
8. **合规第一**：一次违规可能毁掉整个频道
