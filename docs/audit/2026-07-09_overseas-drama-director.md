# 1. 系统层面审阅

## 现在做对了什么

1. **诊断维度够完整**
   - 代码里已覆盖标题、钩子、骨架、赞率、播放分布、SEO、发布频率、OAuth 留存、流量、地域、受众、封面协同、系列化。
   - 这套维度已经接近一个短剧频道运营后台，而不是普通内容分析器。

2. **把 1% 留存和 3 分钟留存放进核心框架是对的**
   - prompt 明确要求有 OAuth 时必须引用 1% 和 3 分钟。
   - 对 60-120 分钟短剧，整体完播率意义有限，前 3 分钟才是算法是否继续测试的关键。

3. **JSON 结构适合产品化**
   - `channel_llm`、`video_scores`、`retention_data`、`summary` 分层清楚。
   - `panel_v3.py` 里 `_merge_video_scores_from_diagnoses()` 保留历史评分，这个设计对追踪优化前后变化有价值。

4. **面板已经开始从“报表”转向“运营工作台”**
   - ChannelAnalysis.vue 展示订阅、播放、留存、观看时长、净增粉、千播订。
   - 这些指标是短剧频道日常复盘必须看的，不是虚指标。

5. **样本诊断能抓住最大伤口**
   - 追劇姐妹：1% 留存 97%，3 分钟留存 23%，平均留存 9.53%。
   - 诊断把“3 分钟断崖”列为 critical，是正确的。

---

## 关键缺陷或盲点

1. **把推荐流量当 CTR 代理，容易误判频道健康**
   - prompt 写“推荐>40%=CTR隐性健康”。
   - 但样本推荐 59%、3 分钟留存 23%、赞率 0.87%。这不是健康，是算法在测试，但内容没接住。
   - 我会把它定义成：**高曝光低兑现，存在推荐回撤风险**。

2. **没有真实 CTR / 曝光量字段，标题封面诊断缺最关键闭环**
   - 现在只能用播放、赞率、推荐占比反推。
   - 但标题封面真正该看：`impressions / CTR / views from browse / cover version`。
   - 没 CTR，封面协同 7.8 分更多是文本判断，不是结果判断。

3. **发布时间没有地区化换算**
   - 样本 `UTC 10点` 占 80%，换算 HKT/TW 是 18:00。
   - HK 黄金是 20:00-22:00，TW 是 21:00-23:00。
   - 系统只说“时段集中”，没指出“发早了 2-4 小时”，这个会直接影响首小时点击密度。

4. **SEO 被识别了，但没有变成强制动作**
   - 样本：`avg_tags_per_video = 0`，搜索流量 0.8%。
   - 这在冷启动频道是 major 甚至 critical，因为推荐一旦停，频道没有长尾入口。
   - action 前 5 没有 SEO 补齐，我不同意。

5. **面板展示还停留在“看数据”，不是“派任务”**
   - UI 展示留存、观看时长、净增粉，但没有告诉运营：今天几点发、发什么题材、哪条换封面、哪条补标签、哪条剪 Shorts。
   - 运营系统的最终价值不是诊断，而是把诊断变成排期和动作。

# 2. 单频道诊断样本审阅：追劇姐妹

## health_score 是否合理？

**5.5/C 基本合理，但我会压到 5.0-5.2。**

理由：

- 优势确实有：1% 留存 97%，推荐流量 59%，说明开头和算法测试都不差。
- 但硬伤太多：
  - 3 分钟留存 23%，低于 30%基准。
  - 平均留存 9.53%，低于短剧健康线 15%-25%。
  - 赞率 0.87%，低于 1%，转化差。
  - 搜索 0.8%，SEO 空白。
  - 标签 0 个。
  - 主题材“重生”仅 20%，一致性 0.2。
  - 头部 3 条占比 50.2%，刚好踩进爆款依赖区。

所以 5.5 不算错，但偏宽松。这个频道不是“中等健康”，是“算法还愿意试，但内容和运营没接住”。

---

## bottleneck 是否合理？

**“中段内容流失”判定对，但不完整。**

更准确应该拆成三层瓶颈：

1. **内容承接瓶颈**
   - 1% 留存 97% → 3 分钟 23%。
   - 说明标题/开头有冲击，但 1-3 分钟没有给足反转、羞辱、打脸、身份揭露。

2. **频道标签瓶颈**
   - 主类型仅 20%，一致性 0.2。
   - YouTube 很难判断你到底是重生、家庭、甜宠还是虐恋，推荐人群不稳定。

3. **流量入口瓶颈**
   - 搜索 0.8%，标签 0。
   - 现在高度依赖推荐和订阅，缺 SEO 长尾，不利于冷启动。

---

## actions 是否合理？

现有 5 条里，前 4 条方向对：

1. 优化 3 分钟内容：对，应该 P0。
2. 去掉“火爆短劇全集”前缀：对。
3. 系列化：对。
4. 封面协同：对。
5. 订阅互动引导：对，但优先级应低于 SEO 和发布时间。

**漏掉 3 个更该排前面的动作：**

- **SEO 补齐**：0 标签、搜索 0.8%，必须进 top3。
- **本地黄金时段**：UTC10=繁中18点，偏早，应该测试 TW 21:00 / HK 20:30。
- **Shorts 引流**：153 订阅，离 1000 还差 847，长视频日更不够，必须每天切 3-5 条爆点 Shorts。

## 被漏掉的更精准视角

1. **“订阅流量 32.6%”不能直接判断粉丝粘性强**
   - 总订阅只有 153。
   - 小基数下，几十个老观众反复看就能拉高订阅流量占比。
   - 应结合 `subscribers / views_30d / returning viewers`，否则会高估粉丝盘。

2. **标题长度建议和频道实测冲突**
   - 系统建议扩到 80-100 字符。
   - 但样本 `≤40字符` 赞率 1.33%，高于 `41-60` 的 0.78% 和 `61-80` 的 0.69%。
   - 正确做法不是一刀切 84 字，而是 A/B：短冲突标题 vs 长 SEO 标题。

3. **2hr+ 视频表现反而优于 1-2hr**
   - 样本：1-2hr 均播 486，赞率 0.74%；2hr+ 均播 764，赞率 1.13%。
   - 这说明该频道受众可能接受长合集，问题不是“太长”，而是前 3 分钟和中段钩子密度。

# 3. 可行性优化方案

## P0｜增加“算法状态”字段，修正推荐流量误判

- **改什么**：`diagnose_channel.py` prompt 的 CTR 代理规则；输出 JSON 新增 `algorithm_status`。
- **为什么**：样本推荐 59%，但 3 分钟留存 23%、平均留存 9.53%、赞率 0.87%。不能写“流量结构健康”，应写“算法测试中但承接失败”。
- **怎么做**：
  ```json
  "algorithm_status": {
    "stage": "testing_not_converting",
    "reason": "recommend_pct>40 but retention_3min<30 and like_rate<1"
  }
  ```
  规则：推荐 >40 且 3 分钟 <30 → 禁止输出“CTR健康”。
- **预期效果**：频道权重误判减少 30%-40%，避免运营继续放大错误内容。
- **优先级**：P0。

---

## P0｜发布时间换算成本地市场，并给出命中率

- **改什么**：`prepare_channel_findings()` 增加 `local_posting_analysis`；Vue 展示“黄金时段命中率”。
- **为什么**：样本 80% 发在 UTC10，即 HKT/TW 18:00，早于 HK 20:00-22:00、TW 21:00-23:00。
- **怎么做**：
  - 根据 `geo_detail.top_regions` 判定主市场。
  - 台湾第一 → 用 TST；香港第一 → 用 HKT；繁中默认 UTC+8。
  - 新增：
    ```json
    "local_posting": {
      "top_local_hour": 18,
      "golden_window": "21:00-23:00",
      "golden_hit_rate": 0
    }
    ```
- **预期效果**：首 2 小时点击密度提升 20%-50%；成熟后 CTR 可提升 1-2 个百分点。
- **优先级**：P0。

---

## P0｜把 SEO 从“问题提示”升级为“强制行动”

- **改什么**：`actions` 生成规则和 prompt 输出要求。
- **为什么**：样本标签 0、搜索 0.8%。冷启动频道不能只靠推荐，SEO 是长尾播放兜底。
- **怎么做**：
  - 若 `avg_tags_per_video < 10` 或 `search_pct < 5`，SEO action 必须进入前 3。
  - 输出固定三件套：
    1. 标题关键词建议。
    2. 描述前 150 字模板。
    3. 15-20 个标签。
  - 繁中标签例：
    `台灣短劇、香港短劇、重生復仇、豪門恩怨、真假千金、打臉爽劇、一口氣看完、短劇全集、霸總護妻、家庭倫理`
- **预期效果**：搜索流量从 0.8% 拉到 5%-10%；长尾播放提升 20%-50%。
- **优先级**：P0。

---

## P1｜3 分钟留存诊断拆成“重剪前 180 秒”任务

- **改什么**：`retention_diagnosis` 增加 `edit_plan`；UI 展示重剪建议。
- **为什么**：现在只说 3 分钟低，运营不知道怎么改。样本 1% 97% 到 3 分钟 23%，流失窗口非常明确。
- **怎么做**：
  ```json
  "edit_plan": [
    "0-3秒直接给羞辱/打脸结果",
    "15秒内出现身份反差",
    "60-90秒放第一次反转",
    "120-180秒必须出现打脸/揭穿/下跪"
  ]
  ```
  若 3 分钟 <30%，强制 action：重剪前 180 秒。
- **预期效果**：3 分钟留存从 23% 提到 30%-35%；AVD 占比从 9.5% 提到 13%-16%。
- **优先级**：P1。

---

## P1｜标题建议改为“频道实测 × 竞品基准”双轨 A/B

- **改什么**：prompt 中“最佳标题长度 84”不要单点输出；新增 `title_ab_plan`。
- **为什么**：样本里 ≤40 字符赞率 1.33%，长标题赞率更低。直接要求 80-100 字，有误伤风险。
- **怎么做**：
  ```json
  "title_ab_plan": {
    "A": "40-55字符，冲突前置",
    "B": "75-90字符，SEO剧情摘要",
    "test_size": "next_6_videos"
  }
  ```
  A 版例：`被丈夫趕出門後，她當眾揭開外遇真相！`
  B 版例：`人妻發現地下室畫作才驚覺丈夫早已出軌，豪門婚姻背後全是陰謀`
- **预期效果**：标题策略准确率提升 25%；CTR 波动降低。
- **优先级**：P1。

---

## P1｜增加“题材纯度评分”，新频道强制收敛 30 天

- **改什么**：`content_consistency` 输出 `genre_focus_score` 和 `next_30d_genre_plan`。
- **为什么**：样本主类型“重生”仅 20%，一致性 0.2。新频道这么混，算法很难打标签。
- **怎么做**：
  - 规则：订阅 <1000 且 primary_ratio <50%，action 必须加入题材收敛。
  - 输出：
    ```json
    "next_30d_genre_plan": ["重生復仇", "豪門打臉"]
    ```
  - 30 天内不发甜宠、悬疑、纯家庭伦理，除非作为副线。
- **预期效果**：推荐人群稳定，均播提升 20%-60%；头部 3 条占比从 50% 降到 35%-40%。
- **优先级**：P1。

---

## P1｜冷启动频道加入 Shorts 引流机制

- **改什么**：`upload_pace` 和 `actions`；Vue 增加“今日 Shorts 任务”。
- **为什么**：追劇姐妹 153 订阅，离 YPP 订阅门槛差 847。只靠长视频自然涨粉太慢。
- **怎么做**：
  - 规则：
    ```python
    if subscribers < 1000:
        shorts_per_day = "3-5"
        long_video_per_day = "1"
    ```
  - Shorts 来源：长视频前 3 分钟、打脸点、身份揭露点、下跪求饶点。
  - 标题用 HK/TW 双版本测试。
- **预期效果**：订阅增长速度提升 30%-80%；Shorts 到长视频转化 3%-8%。
- **优先级**：P1。

---

## P2｜面板从“诊断页”升级为“执行队列”

- **改什么**：ChannelAnalysis.vue 展开详情区新增 3 个模块。
- **为什么**：现在运营看完数据还要自己翻译成动作，执行损耗高。
- **怎么做**：
  1. 今日必做：补标签、换封面、发片时间。
  2. 48 小时复盘：低留存视频、低赞率视频、搜索为 0 视频。
  3. 加倍/放弃：题材、标题骨架、时长区间表现。
- **预期效果**：运营执行效率提升 50%；减少“只看报告不改片”的情况。
- **优先级**：P2。

# 4. 我不同意 distribution-expert 的地方

1. **“冷启动每天 2 部长视频 + 5-8 条 Shorts”我不完全同意**
   - 对追劇姐妹这种 3 分钟留存只有 23% 的频道，先每天 2 部长视频会放大低质量问题。
   - 我的建议是：**长视频 1 条/天 + Shorts 3-5 条/天 + 重剪前 180 秒**。
   - 先把 3 分钟拉到 30% 以上，再加到每天 2 部。

2. **“搜索流量可占 40%”不能作为普遍目标**
   - 短剧频道主流还是推荐和浏览页，搜索 40%更像强 SEO 或剧名 IP 流量。
   - 对这个频道，先从 0.8% 拉到 5%-10%更现实。

3. **“标题扩展到 80-100 字符”不能直接执行**
   - 本频道样本显示 ≤40 字符赞率最高。
   - 我会做 A/B，而不是直接按竞品最佳 84 字改全量标题。
