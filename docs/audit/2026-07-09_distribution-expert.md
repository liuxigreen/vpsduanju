   - channel_llm、video_scores、retention_data、summary 分层清晰。

5. **样本诊断能抓住主瓶颈**

   - 对短剧来说，3分钟留存低于30%就是中段承接失败。

## 关键缺陷或盲点

1. **缺少真实 CTR，CTR 代理过强**
   - prompt 里用“推荐流量>40%=CTR隐性健康”作为代理。
   - 问题：推荐占比高不等于 CTR 高，也可能是系统冷启动测试流量；如果后续留存差，推荐占比仍可能短期偏高。
   - 样本里推荐59%，但留存9.53%、赞率0.87%，不能直接说“流量结构健康”。

2. **SEO 诊断权重不足**
   - 样本搜索仅0.8%，标签0个，但 actions 里 SEO 没进前5。
   - 我们数据里搜索流量可占40%，冷启动第1-14天 SEO 是第二板斧。
   - 现在系统知道 SEO 差，但没有转化为强行动。

3. **发布时区判断错误风险高**
   - 样本 posting_pattern：UTC 10点占80%，对应 HKT/TW 18:00。
   - 诊断只说“分散发布时段”，但短剧繁中黄金时段是20:00-22:00 HKT/TW。
   - 应直接判断：当前比黄金时段早2-4小时，不是简单分散。

4. **频率建议偏保守**
   - 样本订阅153，冷启动期 <1000订阅，规则应是每天2部长视频 + 每天5-8条Shorts。
   - 诊断给“保持日更”，不够激进。
   - 频道40条视频、153订阅，订阅转化瓶颈明显，需要 Shorts 引流而不是只靠长视频日更。

5. **面板展示偏“看数”，缺少“下一步动作”**
   - Vue 前200行核心是指标卡、表格、详情卡。
   - 但没有明显展示：今天发几条、几点发、哪个题材加倍、哪条换封面、哪条加 Shorts。
   - 运营系统不能只诊断，要能生成执行队列。

# 2. 单频道样本审阅：追劇姐妹

## health_score 是否合理？

**5.5/C 基本合理，但略高。**

数据证据：

- 订阅：153，视频40，均播1129，冷启动未失败。
- 推荐流量：59%，说明系统还在给曝光。
- 1%留存：97%，前钩子强。
- 3分钟留存：23%，低于30%基准。
- 平均留存：9.53%，低于短剧60-120分钟健康线15-25%。
- 赞率：0.87%，低于1%，互动转化差。
- 搜索：0.8%，SEO 几乎空白。
- 标签：0个。
- 内容一致性：主类型“重生”仅20%，低于 genre_count <=2 的频道权重原则。

我们会给 **5.0-5.3**。  
理由：推荐占比高是优势，但 SEO、题材一致性、3分钟留存、订阅转化同时弱，综合不应超过5.5太多。

---

## bottleneck 是否合理？

**“中段内容流失”判定合理，但不是唯一瓶颈。**

当前诊断把主瓶颈放在：

> 3分钟留存23%，1-3分钟承接失败。

这是对的。  
但更完整的瓶颈应该是三层：

1. **内容承接瓶颈**
   - 1%处97% → 3分钟23%，说明标题/开头能骗进来，但剧情兑现不足。

2. **分发结构瓶颈**
   - 搜索0.8%、标签0个，长期自然流量入口缺失。
   - 推荐流量59%一旦测试失败，后续没有搜索兜底。

3. **频道定位瓶颈**
   - 主类型仅20%，YouTube 很难给频道打稳定人群标签。
   - 40条视频还没形成“重生/豪门/复仇”单一赛道。

---

## actions 是否合理？

**前3条合理，缺2条关键动作。**

现有 actions：

1. 优化3分钟内容：对。
2. 去掉“火爆短劇全集”前缀：对。
3. 系列化：对。
4. 封面协同：对。
5. 订阅互动引导：对。

漏掉：

1. **SEO 补救动作**
   - 搜索0.8%、标签0个，是 major 级问题。
   - 每条至少15-20个标签，不是样本里“至少5个”。

2. **Shorts 冷启动动作**
   - 当前153订阅，必须每天5-8条 Shorts。
   - Shorts → 长视频转化率约3-8%，这是新频道低权重阶段最现实入口。

3. **黄金发布时间修正**
   - 80%在 UTC 10:00，即繁中18:00。
   - 应改为 HKT/TW 20:30-21:30，固定误差<30分钟。

---

## 被漏掉的更精准视角

1. **“高推荐低留存”不是健康，是算法测试未兑现**
   - 推荐59%不能直接判强。
   - 应标记为“算法测试中，留存未达标，存在推荐回撤风险”。

2. **订阅流量32.6%存在样本基数偏差**
   - 总订阅仅148/153，订阅流量32.6%不一定代表粉丝盘强。
   - 更像小样本老观众反复观看，不能过度解读为“忠实回访”。

3. **标题长度结论冲突**
   - 样本 title_patterns 显示 ≤40字符赞率1.33%最高。
   - 但 LLM 建议扩展到80-100字符。
   - 这里应做“点击/播放/留存”分层，不应只按蒸馏最佳84字符一刀切。

4. **发布时间没有转成本地时间**
   - UTC 10点不能直接说“时段集中”，要换算成 HKT/TW 18:00。
   - 结论应是：偏早，未命中20:00-22:00黄金窗口。

# 3. 可行性优化方案：5-8条

## P0｜加入“真实/代理 CTR 分层”，避免误判推荐健康

- **改什么**：`diagnose_channel.py` prompt 的“CTR代理”和 `traffic_analysis.health`。
- **为什么**：样本推荐59%，但平均留存9.53%、3分钟23%、赞率0.87%。这不是强健康，是“有曝光、低承接”。
- **怎么做**：
  - 字段新增：`ctr_source: real|proxy|missing`。
  - 若无真实 CTR，只输出“CTR代理”，禁止写“CTR健康”。
  - 规则：推荐>40%且3分钟<30% → `algorithm_status="testing_but_not_converting"`。
- **预期效果**：频道权重判断误报减少30%-40%，避免运营误以为推荐结构已经健康。
- **优先级**：P0。

---

## P0｜把发布时段转成本地黄金时段诊断

- **改什么**：`prepare_channel_findings()` 或 `_generate_channel_diagnostics()` 的 `posting_pattern.by_hour`。
- **为什么**：样本80%在 UTC 10点，等于 HKT/TW 18:00，早于20:00-22:00黄金时段。现在只提示“分散测试”，不够准。
- **怎么做**：
  - 根据 language/geo top market 选择时区。
  - 繁中默认 HKT/TW +8。
  - 新增字段：
    ```json
    "local_posting": {"top_hour": 18, "golden_hit_rate": 0}
    ```
  - 命中规则：20-22点为 golden。
- **预期效果**：发布时间命中率从当前0%提升到70%+；同内容播放量可提升1.5-3倍。
- **优先级**：P0。

---

## P0｜SEO 从“诊断项”升级为“行动项”

- **改什么**：prompt 的 actions 要求、`seo_analysis` 诊断规则、面板详情区。
- **为什么**：样本搜索0.8%、标签0个。我们模型里搜索流量可占40%，冷启动第1-14天必须 SEO 截流。
- **怎么做**：
  - 当 `search_pct < 5` 或 `avg_tags_per_video < 10`，强制 action top3 出现 SEO。
  - 标签标准改为15-20个，不是5个。
  - 输出模板：
    - 标题关键词：短劇/全集/繁體/重生/豪門/復仇
    - 描述前两行模板
    - 15个标签列表
- **预期效果**：搜索流量从0.8%提升到5%-10%；长尾自然播放提升20%-50%。
- **优先级**：P0。

---

## P1｜冷启动策略加入 Shorts 任务队列

- **改什么**：`channel_llm.actions`、面板 `ChannelAnalysis.vue` 详情卡增加“今日发布任务”。
- **为什么**：样本153订阅，典型<1000冷启动频道。当前只建议长视频日更，缺少每天5-8条Shorts。
- **怎么做**：
  - 规则：
    ```python
    if subscribers < 1000:
        recommended_long = "daily_2"
        recommended_shorts = "5-8/day"
    ```
  - UI 显示：
    - 今日长视频：2条，20:30/21:30
    - Shorts：5条，截取前3分钟爆点/打脸/身份揭露
- **预期效果**：Shorts→长视频转化3%-8%；订阅增长速度提升30%-80%。
- **优先级**：P1。

---

## P1｜把“3分钟留存”拆成可执行剪辑诊断

- **改什么**：`retention_diagnosis` 输出结构。
- **为什么**：现在只说“3分钟低”，但运营不知道改哪一段。样本1% 97%到3分钟23%，流失发生在1-3分钟。
- **怎么做**：
  - 新增字段：
    ```json
    "retention_drop_window": "1%-3min",
    "edit_fix": [
      "0-15秒直接给冲突结果",
      "60-90秒插入第一次反转",
      "120-180秒必须出现打脸/身份揭露"
    ]
    ```
  - 若3分钟<30%，action 固定为“重剪前180秒”。
- **预期效果**：3分钟留存从23%拉到30%+；AVD占比从9.5%拉到15%左右。
- **优先级**：P1。

---

## P1｜修正标题长度建议：按频道实测而非固定84字符

- **改什么**：prompt 中“最佳标题长度84”的使用方式。
- **为什么**：样本里 ≤40字符赞率1.33%最高，41-60为0.78%，61-80为0.69%。直接建议80-100字符，与本频道数据冲突。
- **怎么做**：
  - 标题建议分两层：
    1. 本频道实测最佳：≤40。
    2. 竞品蒸馏最佳：84。
  - 输出为 A/B 测试：
    - A版：40-55字符，高冲突短标题。
    - B版：75-90字符，剧情摘要型SEO标题。
- **预期效果**：标题建议准确率提升25%；避免盲目拉长导致 CTR 下滑。
- **优先级**：P1。

---

## P1｜频道定位增加“题材纯度评分”和强制收敛建议

- **改什么**：`content_consistency` 与 `channel_weight`。
- **为什么**：样本主类型“重生”仅20%，一致性0.2。新频道前3个月不应跨题材，genre_count 应≤2。
- **怎么做**：
  - 新增：
    ```json
    "genre_focus_score": 0.2,
    "allowed_genres_next_30d": ["重生逆襲", "豪門復仇"]
    ```
  - 若 primary_ratio < 0.5，行动清单强制加入“连续30天只发一个题材”。
- **预期效果**：推荐人群更稳定；同题材视频均播提升20%-60%。
- **优先级**：P1。

---

## P2｜面板从“诊断展示”升级为“运营执行台”

- **改什么**：`ChannelAnalysis.vue` 展开详情区。
- **为什么**：现在展示很多数，但没有下一步排期、换封面清单、SEO补齐清单。运营每天仍需人工转译报告。
- **怎么做**：
  - 增加三个模块：
    1. 今日必做：发片时间、Shorts数量、SEO补标签。
    2. 48小时复盘：低CTR/低留存视频列表。
    3. 加倍/放弃：爆点得分公式。
  - 爆点得分：
    `CTR × 完播率 × AVD / 100`，无CTR时用推荐代理标记。
- **预期效果**：运营执行效率提升50%；减少“看完报告不行动”的问题。
- **优先级**：P2。
